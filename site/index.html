What Talos does today

Visible work before local changes.

Talos handles bounded workspace tasks: inspect a project, retrieve local context, create and edit files after approval, run bounded command profiles, and report checked outcomes with trace evidence.

It supports approved file creation and edits, approved workspace operations, bounded command profiles, prompt-debug evidence, and runtime-owned outcomes instead of model-authored victory prose.

Trust boundaries

Control surfaces are visible before action.

Talos should never depend on the model sounding confident. The runtime owns approval, tool exposure, result checks, protected reads, unsupported file handling, and final mutation truth.

Trust boundaries

Workspace files Read-only inspection first

Talos starts by inspecting scoped workspace files before mutation tools are exposed.

File writes Preview and approval required

File creation and edits are approved before the workspace is changed.

Workspace operations Approved before mkdir, copy, move, rename, or delete

Directory, file movement, and destructive tools are treated as workspace mutations.

Command profiles Bounded and approved before execution

Command execution is routed through configured profiles and approval gates.

Unsupported binary documents Refused instead of faked

Talos does not pretend a text write is a valid PDF, DOCX, or unsupported binary file.

Default posture: selected workspace, local engine, approved mutation, checked outcome.

Local model setup

Managed llama.cpp is the beta path.

The current product path uses managed llama.cpp with tested Qwen and GPT-OSS profiles. Talos can place Hugging Face model files under ~/.talos/models/huggingface on first start, or use a user-owned GGUF path.

Ollama remains available as a legacy backend, but the landing page should lead with the engine path Talos is hardening now.

Qwen coder profile

talos setup models --profile qwen2.5-coder-14b --server-path C:/path/to/llama-server.exe --write

GPT-OSS profile

talos setup models --profile gpt-oss-20b --server-path C:/path/to/llama-server.exe --write
